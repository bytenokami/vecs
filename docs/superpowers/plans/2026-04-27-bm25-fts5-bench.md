# BM25 → FTS5 A/B Benchmark Results

**Date:** 2026-04-27
**Corpus:** `bloomly-code` collection — 750 chunks
**Hardware:** Local dev (macOS, Python 3.13, sqlite_version 3.53.0)

## Build phase

| Operation                  |   Time |
|----------------------------|-------:|
| legacy (rank_bm25 build)   | 143 ms |
| fts5   (build full)        | 181 ms |
| fts5   (single upsert)     | 0.33 ms |
| fts5   (single delete)     | 0.08 ms |

The full build is ~25 % slower under FTS5 (181 vs 143 ms) — SQLite write overhead vs in-memory list construction. That cost is paid **once** at first run; from then on, the indexer only pays the **incremental** upsert/delete cost (sub-millisecond per chunk), which is the entire point of the migration. Under the prior `_rebuild_bm25`, a 1-line edit triggered the full 143 ms rebuild; under `_sync_bm25`, the same edit is ~0.33 ms — a **~430× speedup** at the per-tick hot path.

## Summary

- **mean top-5 overlap:  0.97**
- **mean top-10 overlap: 0.97**
- legacy median query: 0.23 ms
- fts5   median query: 0.16 ms

## Acceptance

The plan documented an acceptance threshold of "mean top-10 overlap ≥ 0.5 (tune column weights or revisit if below)." Actual: **0.97**. Result quality is preserved; the one query with <1.0 overlap (`tokenize camelCase`) shows a tie-breaker shuffle at rank ≥4, not a meaningful recall regression.

Query latency is similar at 750 chunks (legacy slightly faster up to rank-5 due to in-memory access, FTS5 slightly faster on broader queries via inverted-index pruning). At the scale of 42K+ chunks the legacy in-memory `BM25Okapi.get_scores` becomes O(N) per call while FTS5 stays sub-linear; the larger the corpus, the bigger the FTS5 win.

## Net win

The migration trades a one-time +38 ms full-build cost for:
1. **Incremental indexing** — per-tick BM25 maintenance drops from O(N tokenize + N pickle) to O(diff size) for SQL writes.
2. **No third-party dependency** — `rank-bm25` removed from `pyproject.toml`.
3. **Atomic durability** — SQLite transactions + WAL replace pickle's "atomic-rename-not-implemented" risk.
4. **Bounded memory** — the prior `BM25Okapi` held the entire tokenized corpus in process RAM; FTS5 streams from disk.
