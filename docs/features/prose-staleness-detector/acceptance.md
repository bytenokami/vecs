- [x] **Row schema** in `<project>-prose-facts`: `id` = UUID4 per row; `embedding` = `voyage(f"{subject} {predicate} {object}")`; `document` = `f"{subject} {predicate} {object}"`; `metadata` carries `subject`, `predicate`, `object`, `chain_key = f"{subject}|{predicate}"`, `valid_from: int` (unix epoch ms), `invalid_at: int` (`0` sentinel = "still current"; non-zero ms = invalidation timestamp; Chroma metadata does not accept `None`), `is_current: bool` (companion field; `True` for the one current row in a chain, `False` on superseded rows; field name + bool-vs-int representation is pinned post-Phase-7-dry-run per Fix 1 ÔÇö if Chroma normalizes bool to int, swap to `is_current_int: 0|1`), `source_id: str` (session id), `version: int` (starts at 1, bumps by 1 per SUPERSEDE). - [x] **Row-id strategy**: UUID4 per row + `chain_key` metadata column (not `subject|predicate#vN` suffix scheme). Lookup of current state uses `collection.get(where={"chain_key": ..., "is_current": True})`. Documented constraint: Chroma's `where` clause cannot match `invalid_at IS NULL` directly, so the `is_current: bool` companion field is the operative filter.
- [x] **State machine semantics**: per extracted session-side triple `(subject, predicate, object)`:
  - Lookup `where={"chain_key": ..., "is_current": True}`. If multiple rows return (transient SUPERSEDE-recovery state per Fix 2), the highest-version row is operative; lower-version rows are repaired by flipping `is_current` to `False` and setting `invalid_at=now_ms`.
  - Lookup empty (after repair) Ôćĺ **INSERT** new row with `valid_from=now_ms`, `invalid_at=None`, `is_current=True`, `version=1`. Function returns `EVENT_INSERT`.
  - Lookup found with same `object` Ôćĺ **NOOP**. No write. Returns `EVENT_NOOP`.
  - Lookup found with different `object` Ôćĺ **SUPERSEDE (crash-safe reorder, Fix 2)**. Writes are reordered to make the transition crash-recoverable: (a) `collection.add(...)` the new row with `valid_from=now_ms`, `invalid_at=None`, `is_current=True`, `version=old_version+1` FIRST; (b) `collection.update(...)` the old row with `invalid_at=now_ms`, `is_current=False` SECOND. If (a) succeeds and (b) fails, two rows briefly carry `is_current=True`; the next-lookup repair branch restores the invariant by selecting the highest-version row and demoting the older one. Returns `EVENT_SUPERSEDE`.
- [x] Event names are module-level constants: `EVENT_INSERT = "INSERT"`, `EVENT_NOOP = "NOOP"`, `EVENT_SUPERSEDE = "SUPERSEDE"` in `src/vecs/prose_drift.py`. Asserted in unit test.
- [x] **No deletes.** No code path issues `collection.delete(...)` against `<project>-prose-facts`. Grep test asserts this. SUPERSEDE is append + flag-flip only. History is queryable forever via `valid_from`/`invalid_at` filters.

## stage-2 recall ÔÇö cross-predicate / paraphrase

Source: `docs/features/prose-staleness-detector/stage2-recall-design.md`. Closes the v1 exact-`chain_key` boundary via an embedding-similarity fallback + ONE Opus contradiction-judge on MISS. Top-1 candidate, always-on, bounded by similarity threshold + verdict cache.

- [x] `_cosine(a, b)` returns standard cosine similarity; a zero-norm vector yields `0.0` (no `ZeroDivisionError`). Unit-tested at identical/orthogonal/opposite/zero reference points.
- [x] `_load_current_rows(collection)` returns `[(meta, embedding)]` for every `is_current=True` fact, loaded once per scan, deduped to the max-version row per `chain_key`, read-only (does not flip a transient duplicate `is_current` row).
- [x] `_best_semantic_candidate(doc_emb, rows)` returns the highest-cosine `(meta, sim)`; `None` when `rows` is empty. Threshold gating is the caller's responsibility.
- [x] On a `chain_key` MISS, `find_prose_drift` embeds the doc triple, takes the single best `is_current` candidate, and escalates to the judge only when `sim >= STAGE2_SIM_THRESHOLD` (0.85). Below threshold Ôćĺ no judge call, no drift.
- [x] Judge model id is exactly `claude-opus-4-7` (module constant `PROSE_JUDGE_MODEL`); judge calls pass no `temperature`. Judge returns `{contradicts, confidence, reason}`; fence-tolerant parse.
- [x] Judge verdicts cached in `judge_cache(doc_triple_json, chat_triple_json, model, prompt_version, verdict_json)`; a repeat scan with no new facts/docs makes zero new anthropic calls (rerun-determinism preserved).
- [x] A semantic finding emits a drift entry with `match_type="semantic"`, `similarity`, `confidence`, and a `chat` block carrying the chat-side `subject`/`predicate`/`object`/`session_id` (which differ from the doc-side `subject`/`predicate`). Exact findings gain `match_type="exact"` and keep their v1 shape.
- [x] Judge says NOT a contradiction (above threshold) Ôćĺ no drift entry.
- [x] A single judge call error (API or unparseable) is caught: that candidate is skipped, the scan continues, and `report["stage2_judge_errors"]` counts it. `report["stage2_judge_calls"]` counts judge invocations.
- [x] The cross-predicate scenario (chat `team|employs:sasha` vs doc `team|has_role:"no backend developer"`) now surfaces exactly one semantic drift ÔÇö `test_cross_predicate_paraphrase_drift_is_detected` is promoted from `xfail(strict)` to a passing test.
- [x] CLI prints semantic lines with both predicates and a `[semantic sim=.. conf=..]` tag; the trailing note states cross-predicate/paraphrase is now partially covered while omission and soft/temporal contradictions remain out of scope.
- [x] All pre-existing prose-drift tests stay green (exact path unchanged in behaviour).
